## üá∫üá∏ English

**AI Studio Local Bridge** is a developer tool that connects Google AI Studio (Gemini) directly to your local filesystem. It eliminates the tedious need to manually copy-paste code files generated by the LLM.

It allows Gemini to **create**, **edit**, and **manage** files in your local projects (Godot, Node.js, Python, Java, etc.) directly from the browser interface.

### ‚ú® Key Features

*   **‚ö° One-Click Sync:** Automatically detects code blocks in the latest AI response and saves them to disk.
*   **üìÇ Dynamic Workspace:** Switch between different projects instantly via the browser UI. The server remembers your project history.
*   **üéÆ Game Engine Ready:** Automatically converts `res://` paths (Godot) to valid local paths.
*   **üõ°Ô∏è Safety First:**
    *   **Automated Backups:** Every file overwrite is backed up to `.ai-bridge/backups/` inside your project before changes are applied.
    *   **Sandboxing:** Prevents writing files outside the designated project root.
    *   **Smart Parsing:** Uses an "Edit Icon Fence" strategy to strictly ignore chat history and only sync the latest response.

### üöÄ Installation

#### 1. Server Setup (Node.js)
The server runs locally and handles file operations.

```bash
cd server
npm install
```

#### 2. Client Setup (Browser)
1.  Install the **Tampermonkey** extension for your browser (Chrome/Edge/Firefox).
2.  Create a new script in Tampermonkey.
3.  Copy the content of `client/ai-studio-bridge.user.js` into the script editor.
4.  Save the script.

### üõ†Ô∏è Usage

#### 1. Start the Server
Run the bridge server. It will listen on port `3000`.

```bash
# Inside /server directory
node src/index.js
```

#### 2. Configure AI Studio
1.  Open [Google AI Studio](https://aistudio.google.com/).
2.  You will see the **AI WORKSPACE** panel in the bottom-right corner.
3.  Enter the **absolute path** to your local project (e.g., `C:/Projects/MyGame` or `/Users/dev/cool-app`) in the input field and click **SET**.
4.  The indicator will turn green.

#### 3. Prompting (Important!)
For the bridge to detect files correctly, the LLM must output the file path immediately before the code block.
**Add this to your System Instructions:**

> **CODING RULE:**
> Always provide the file path immediately before the code block using the `###` header:
>
> ### `path/to/file.ext`
> ```language
> // code here...
> ```
> For Godot projects, use `res://path/to/script.gd`.

#### 4. Syncing
1.  Ask the model to write code.
2.  The Bridge panel will detect the files and show a "SYNC TO DISK" button.
3.  Click it. The files are now on your hard drive! üöÄ
